## 项目一：服务器自动化运维脚本工具箱

**项目描述**：
开发了一套 Shell 自动化运维脚本工具箱，覆盖服务器巡检、日志分析、数据备份、批量管理四大核心运维场景，实现日常运维操作的自动化，提升运维效率。

**技术栈**：
Shell、Linux 系统命令、awk/sed/grep、Crontab、SSH、mysqldump

**主要工作**：

1. **系统巡检模块**：实现 CPU/内存/磁盘/端口/进程的自动检测，支持阈值告警，通过 Crontab 每小时自动执行，替代人工巡检

2. **日志分析模块**：使用 awk/sed/grep 解析 Nginx 访问日志，自动统计 TOP10 IP、HTTP 状态码分布、热门接口，生成可视化分析报告

3. **数据备份模块**：实现 MySQL 全量定时备份，采用 mysqldump + gzip 压缩方案，支持自动校验备份完整性和过期备份清理

4. **批量管理模块**：基于 SSH 免密登录实现多主机批量命令执行，支持配置文件管理主机列表，提升批量运维效率

**项目成果**：
- 将日常巡检时间从 30 分钟/次降低到自动化执行
- 备份策略实现 7 天自动轮转，确数据安全
- 批量操作效率提升 80% 以上

## 面试话术参考

**项目介绍**：
> 这个项目是我在学习 Linux 运维时开发的一套自动化脚本工具箱。主要解决运维工作中频繁重复的操作问题，包括服务器巡检、日志分析、数据备份和批量管理四个模块。通过 Shell 脚本和 Crontab 定时任务，实现了这些操作的自动化执行。

**技术亮点**：
> 在日志分析模块中，我使用了 awk、sed、grep 这些文本处理工具来解析 Nginx 日志，通过管道操作组合多个命令，实现了高效的日志分析。在备份模块中，采用了 mysqldump 的 single-transaction 参数来保证 InnoDB 的一致性备份，避免锁表。

**遇到的挑战**：
> 最初备份脚本没有做文件校验，有一次备份文件损坏但没有发现。后来加上了文件大小校验和返回值检查，确保备份的可靠性。这让我意识到自动化脚本中异常处理的重要性。
