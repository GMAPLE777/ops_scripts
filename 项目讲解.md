# 项目一：服务器自动化运维脚本工具箱 - 详解文档

## 一、项目背景

在日常运维工作中，服务器巡检、日志分析、数据备份、批量管理等操作频繁且重复。人工执行不仅效率低，还容易遗漏或出错。本项目通过 Shell 脚本将这些操作自动化，提升运维效率和可靠性。

## 三、模块详解

### 3.1 系统巡检模块 (check_system.sh)

#### 功能说明
自动检测服务器的 CPU、内存、磁盘、端口、进程状态，当指标超过阈值时输出告警。

#### 核心实现

**CPU 使用率检测**
```bash
cpu_usage=$(top -bn1 | grep "Cpu(s)" | awk '{print $2}' | cut -d. -f1)
```
- `top -bn1`: 批量模式运行一次，获取 CPU 信息
- `awk '{print $2}'`: 提取第二列（用户态 CPU 使用率）
- `cut -d. -f1`: 取整数部分

**内存使用率检测**
```bash
mem_total=$(free -m | awk 'NR==2{print $2}')
mem_used=$(free -m | awk 'NR==2{print $3}')
mem_usage=$((mem_used * 100 / mem_total))
```
- `free -m`: 以 MB 为单位显示内存信息
- `NR==2`: 取第二行（Mem 行）
- 算术计算得出使用率百分比

**磁盘使用率检测**
```bash
df -h | grep -v tmpfs | grep -v loop | awk 'NR>1{print $5 " " $6}' | while read rate mount; do
    num=${rate%\%}
    [ $num -gt $DISK_WARN ] && log "【告警】${mount} 磁盘使用率超过阈值"
done
```
- `grep -v tmpfs`: 过滤临时文件系统
- `${rate%\%}`: 去掉百分号，提取数字

**端口存活检测**
```bash
ss -tuln | grep -q ":$port "
```
- `ss -tuln`: 查看监听的 TCP/UDP 端口
- `grep -q`: 静默模式，只返回状态码

**进程存活检测**
```bash
pgrep -x "$proc" > /dev/null
```
- `pgrep -x`: 精确匹配进程名
- 返回 0 表示进程存在

#### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| CPU_WARN | 80 | CPU 告警阈值 (%) |
| MEM_WARN | 80 | 内存告警阈值 (%) |
| DISK_WARN | 85 | 磁盘告警阈值 (%) |
| CHECK_PORTS | 80, 3306, 9090 | 检测端口列表 |
| CHECK_PROCESSES | nginx, mysqld | 检测进程列表 |

### 3.2 Nginx 日志分析模块 (analyze_nginx_log.sh)

#### 功能说明
解析 Nginx 访问日志，统计总请求量、TOP10 IP、HTTP 状态码分布、TOP10 接口。

#### Nginx 日志格式
```
192.168.1.100 - - [23/Jun/2026:10:00:00 +0800] "GET /api/users HTTP/1.1" 200 1234
```
字段位置：$1=IP, $7=接口, $9=状态码

#### 核心实现

**TOP10 IP 统计**
```bash
awk '{print $1}' $LOG_PATH | sort | uniq -c | sort -nr | head -10
```
- `awk '{print $1}'`: 提取第一列（IP 地址）
- `sort | uniq -c`: 排序后统计重复次数
- `sort -nr`: 按数字逆序排列

**HTTP 状态码分布**
```bash
awk '{print $9}' $LOG_PATH | sort | uniq -c | sort -nr
```
- 提取第九列（状态码）
- 统计各状态码出现次数

**TOP10 接口统计**
```bash
awk '{print $7}' $LOG_PATH | sort | uniq -c | sort -nr | head -10
```
- 提取第七列（请求接口）
- 统计最热门接口

#### 输出示例
```
===== Nginx 访问日志分析报告 =====
生成时间：2026-06-23 10:00:00
日志文件：/var/log/nginx/access.log

1. 总请求量：
  总请求数：156789

2. 访问量 TOP10 IP：
  第1名：192.168.1.100，请求2345次
  第2名：10.0.0.50，请求1890次
  ...

3. HTTP状态码分布：
  状态码200：145678次
  状态码404：8900次
  状态码500：2211次
  ...
```

### 3.3 数据备份模块 (backup_mysql.sh)

#### 功能说明
自动备份 MySQL 数据库，压缩存储，定期清理过期备份。

#### 核心实现

**全量备份**
```bash
mysqldump -u$DB_USER -p$DB_PASSWORD --all-databases --single-transaction | gzip > $BACKUP_DIR/mysql_all_$DATE.sql.gz
```
- `--all-databases`: 备份所有数据库
- `--single-transaction`: InnoDB 一致性备份，不锁表
- `gzip`: 压缩备份文件

**备份校验**
```bash
if [ $? -eq 0 ] && [ -s $BACKUP_DIR/mysql_all_$DATE.sql.gz ]; then
    echo "备份成功"
fi
```
- `$?`: 上一条命令返回值
- `-s`: 文件存在且大小大于 0

**自动清理**
```bash
find $BACKUP_DIR -name "mysql_all_*.sql.gz" -mtime +$RETENTION_DAYS -delete
```
- `-mtime +7`: 修改时间超过 7 天
- `-delete`: 删除匹配文件

#### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| DB_USER | root | 数据库用户名 |
| DB_PASSWORD | your_password | 数据库密码 |
| RETENTION_DAYS | 7 | 备份保留天数 |

### 3.4 批量管理模块 (batch_cmd.sh)

#### 功能说明
通过 SSH 在多台主机上批量执行命令，减少重复登录操作。

#### 前置条件：SSH 免密登录

```bash
# 1. 生成密钥对
ssh-keygen -t rsa

# 2. 分发公钥到目标主机
ssh-copy-id root@192.168.1.101
ssh-copy-id root@192.168.1.102
```

#### 核心实现

```bash
for host in $(cat $HOST_FILE); do
    echo "===== 在 $host 上执行命令 ====="
    ssh root@$host "$CMD"
done
```
- 读取主机列表文件
- 遍历每台主机执行 SSH 命令

#### 使用示例

```bash
# 查看所有主机的主机名
bash batch_cmd.sh "hostname"

# 查看所有主机的磁盘使用
bash batch_cmd.sh "df -h"

# 批量重启 nginx
bash batch_cmd.sh "systemctl restart nginx"
```

## 四、定时任务配置

### 4.1 Crontab 语法

```
┌───────────── 分钟 (0-59)
│ ┌───────────── 小时 (0-23)
│ │ ┌───────────── 日期 (1-31)
│ │ │ ┌───────────── 月份 (1-12)
│ │ │ │ ┌───────────── 星期 (0-7, 0和7都是周日)
│ │ │ │ │
* * * * * command
```

### 4.2 本项目定时任务

```bash
# 每天凌晨2点执行MySQL备份
0 2 * * * /bin/bash /opt/ops_scripts/bin/backup_mysql.sh

# 每小时执行一次系统巡检
0 * * * * /bin/bash /opt/ops_scripts/bin/check_system.sh

# 每天凌晨1点生成Nginx日志分析报告
0 1 * * * /bin/bash /opt/ops_scripts/bin/analyze_nginx_log.sh
```

### 4.3 部署步骤

```bash
# 编辑定时任务
crontab -e

# 粘贴以上配置内容

# 查看已配置的定时任务
crontab -l

# 检查定时任务执行日志
grep CRON /var/log/syslog
```

## 六、常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 脚本执行报错 | 路径错误或命令未安装 | 检查路径、安装依赖 |
| MySQL 备份失败 | 账号密码错误或权限不足 | 检查 mysqldump 可用性 |
| 定时任务不执行 | crond 未启动或路径错误 | `systemctl status crond` |
| SSH 免密失败 | 权限问题 | 检查 `.ssh` 目录权限为 700 |
| 日志分析无输出 | Nginx 日志路径错误 | 确认日志文件存在 |

## 七、技术要点总结

1. **文本处理三剑客**: awk、sed、grep 的熟练使用
2. **Shell 变量**: 数组、字符串处理、算术运算
3. **条件判断**: 数值比较、文件测试、命令返回值
4. **循环控制**: for 循环遍历数组和文件
5. **管道操作**: 多命令组合处理数据流
6. **定时任务**: Crontab 配置和管理
7. **SSH 免密**: 密钥生成和分发
